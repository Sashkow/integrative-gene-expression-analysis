## Pipeline steps and gene counts

The `pipeline_restored()` function in `run.R` takes limma DE results through
STRING PPI network analysis to produce `difexp_final.csv`.

### Step-by-step with gene counts

| Step | Description | 1_2 | 2_3 |
|------|-------------|-----|-----|
| 1. Input | `difexp.tsv` — all limma DE genes (adj.P.Val < 0.05) | 3912 | 6926 |
| 2. logFC filter | Keep only \|logFC\| >= 1 | 326 | 595 |
| 3. STRING mapping | Map gene SYMBOLs to STRING identifiers | 308 | ~560 |
| 4. PPI network | Build subnetwork, remove isolated nodes | 305 | ~540 |
| 5. Clustering | Fastgreedy community detection | 305 | ~540 |
| 6. Centrality | Betweenness centrality per node | 305 | ~540 |
| 7. Annotation | Add is_secreted from HPA | 305 | ~540 |
| 8. Output | `difexp_final.csv` (original) | 268 | 536 |

Note: 1_2 original has 268 rows (not 305) because the original pipeline used
`takeFirst=FALSE` in STRING mapping, which can produce multiple STRING_id entries
per gene (e.g. RNASE4 maps to 2 STRING isoforms), but then some are lost as
isolated nodes. The final count depends on network topology.

### How STRING mapping works

STRING mapping converts human gene symbols (HGNC) to STRING protein identifiers
(e.g. `SYMBOL "LEP"` -> `STRING_id "9606.ENSP00000252951"`).

1. **Download aliases**: The pipeline downloads `9606.protein.aliases.v11.0.txt.gz`
   from STRING (v11), which contains ~20M alias-to-protein mappings for human.

2. **Match by SYMBOL**: Each gene's SYMBOL is looked up in the alias table.
   The mapping uses `multi_map_df()` which does case-insensitive matching
   against all alias types (gene name, UniProt ID, Ensembl ID, etc.).

3. **One-to-many mappings**: With `takeFirst=FALSE`, a single gene symbol can
   map to multiple STRING proteins (isoforms). Duplicate STRING_ids are removed,
   but the same gene can appear on multiple rows with different STRING_ids.

4. **Background mapping**: The full ComBat-corrected expression matrix
   (`sub_exprs.tsv`) is also mapped to STRING. This background (16,765 proteins
   for 1_2; ~13,700 for 2_3) is used by STRING for enrichment significance
   calculations.

5. **Unmapped genes**: ~6% of DE genes and ~17% of background genes cannot be
   mapped. These are typically non-coding RNAs, pseudogenes, or genes without
   STRING protein entries. The pipeline saves these to `unmapped_genes.csv`.

### PPI network filtering

After STRING mapping, a protein-protein interaction (PPI) subnetwork is built
from the mapped DE genes using STRING's combined interaction score
(threshold >= 100, very permissive). Proteins with zero interactions (isolated
nodes) are removed — these are genes that have STRING entries but no known
interactions with other DE genes at the given confidence threshold.

## How the supplement was created from difexp_final

`lykhenko_Supplement1_sorted_include_present.xlsx` sheets 1 and 2 are subsets of difexp_final:
- 1_2: kept clusters 1-4 (253 of 268 rows), dropped clusters 5-9
- 2_3: kept clusters 1-2,4-8 (489 of 536 rows), dropped cluster 3 and clusters 9-14, renumbered sequentially

## Upstream pipeline (merge + ComBat + limma)

The input `difexp.tsv` and `sub_exprs.tsv` were created by:
- `scripts/unite.R` - merge datasets, ComBat batch correction with `~trim_term + Combined.Fetus.Sex`
- `scripts/diff_exp.R` or `r-affymetrix-integration-pipeline/scripts/integrate_example.R` - limma DE

Model: `~0 + trim_term + Combined.Fetus.Sex`, contrast `ii-i` (Second-First) or `iii-ii` (Term-Second).
Filtering: `adj.P.Val < 0.05`, `|logFC| > 1`.
Datasets excluded: GSE55439.
